GAIT: GraphQL API Interactive Tool

Authors: Ungureanu Delia-Elena, Untu George-Adrian

Affiliation: A&D Solutions

Abstract: This report presents GAIT, a web-based tool that enables users to interact with GraphQL APIs using natural language queries. GAIT simplifies API querying by supporting text and voice input, multi-lingual capabilities, and seamless API integration.

1. Introduction

GAIT (GraphQL API Interactive Tool) is designed to bridge the gap between users and complex GraphQL APIs by allowing interaction through natural language constructs. The tool leverages NLP, voice recognition, and real-time API introspection.

2. Related Work

Several tools and platforms exist for interacting with GraphQL APIs, but they often require technical expertise. Notable related work includes:

  • GraphiQL: A web-based interactive GraphQL IDE allowing developers to test and validate queries.
  • ColloQL: A system enabling natural language-based SQL queries, which serves as inspiration for GAIT's query processing pipeline.
  • SPARQL Query Builders: Various RDF-based query builders exist, but few integrate real-time NLP processing for dynamic query generation.
  • Voice-Driven Assistants: Platforms such as Google Assistant and Alexa provide voice-enabled querying, but their integration with GraphQL is minimal.

3. System Architecture

GAIT follows a service-oriented architecture consisting of:

  • Frontend: A React-based SPA for UI interaction.
  • Backend: A FastAPI service handling query processing.

    • The backend of GAIT is built on FastAPI, leveraging modern Python-based web frameworks to provide a high-performance API service. It consists of multiple components:

    • GraphQL Query Engine: Processes natural language inputs, maps them to GraphQL queries, and executes them against various APIs.
    • SPARQL Query Engine: Stores executed queries in RDF format and allows retrieval via SPARQL queries.
    • NLP Processing: Uses SpaCy and deep-translation to convert user inputs into structured queries.
    • Data Persistence: Queries and responses are stored in RDF format to ensure semantic interoperability.
  • APIs: Google Speech-to-Text, Google Translate and GraphQL endpoints.

4. Backend Query Services

The backend services of GAIT handle various aspects of GraphQL query processing, SPARQL data management, and multilingual support. It is designed to efficiently manage and process user queries, convert natural language into structured API requests, and store these interactions for future reference.

4.1 GraphQL Query Processing

GAIT enables users to interact with multiple GraphQL APIs by converting natural language queries into structured GraphQL requests. The backend follows these steps:

  • Query Interpretation: User input is processed using NLP models to extract key entities.
  • API Mapping: The system identifies the relevant GraphQL API based on extracted entities.
  • Query Construction: Dynamic GraphQL queries are generated based on user intent.
  • Execution & Response: Queries are executed, and the JSON response is structured for readability.

This approach allows seamless interaction with multiple GraphQL APIs, providing users with accurate and real-time results.

4.2 SPARQL Query Storage

GAIT incorporates an RDF-based query storage system using SPARQL, which allows structured representation of user queries. The benefits of this approach include:

  • Permanent storage of user queries with metadata.
  • Efficient retrieval of past queries using SPARQL queries.
  • Integration with Linked Data applications for extended interoperability.

The system ensures that user interactions are logged in a format that supports efficient querying and analysis of API usage trends.

4.3 Multilingual Query Support

GAIT provides multilingual query support, making it accessible to users in different linguistic regions. This feature is implemented through:

  • Language Detection: The backend automatically detects the language of the query using DeepL.
  • Translation: If the detected language is not English, the query is translated into English before processing.
  • Response Handling: Query results are translated back into the user’s preferred language.

Currently, GAIT partially supports Romanian, German, and English, with future plans to extend support for additional languages.

4.4 Backend Infrastructure & Performance Optimization

GAIT’s backend is built on FastAPI, ensuring high-performance processing of API queries. Several optimizations have been implemented:

  • Asynchronous Processing: FastAPI’s async capabilities allow non-blocking query execution.
  • Caching: Frequently accessed API responses are cached to reduce redundant requests.
  • Load Balancing: Requests are distributed efficiently to ensure system stability under high loads.
  • Database Optimization: The RDF store is indexed to allow fast retrieval of stored queries.

These enhancements enable GAIT to handle a high volume of user interactions with minimal response time.

5. Implementation

The tool consists of a modular backend using FastAPI and GraphQL processing via gql. The API integrator supports GraphQL, SPARQL, and RDF query handling.

The backend services handle natural language query conversion, API interaction, and RDF storage. The key implementations include:

  • Query Parsing: The NLP engine processes user queries, extracts entities, and maps them to GraphQL-compatible structures.
  • GraphQL Execution: The parsed queries are sent to public GraphQL APIs (e.g., Country API, AniList API) using HTTP clients.
  • SPARQL Query Storage: Every query and its response are stored in an RDF triple store, allowing structured queries via SPARQL.
  • Multi-Language Support: Uses translation APIs to support multiple languages, dynamically translating queries to English.

6. Testing & Evaluation

GAIT has undergone extensive unit and integration testing using pytest. The results indicate high accuracy in query conversion and reliable API interactions.

The backend has been tested rigorously using automated unit tests and integration tests.

  • Unit Testing: Pytest-based tests cover query conversion, API interactions, and RDF storage validation.
  • Integration Testing: Simulated requests are run against public GraphQL APIs to verify response consistency.
  • Performance Testing: Query execution time is measured under different workloads to ensure scalability.
  • SPARQL Query Validation: Tests ensure that stored RDF data is retrievable with accurate SPARQL queries.

7. Challenges & Solutions

Key challenges include handling ambiguous user queries and ensuring multilingual support. Solutions involve NLP enhancements and dynamic query introspection.

8. Future Work & Conclusion

Future enhancements include expanding API support, optimizing NLP models, and improving query validation mechanisms.
